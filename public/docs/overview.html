Gym Management App - Technical Overview

Version 1.0

Tech Stack

Next.js (App Router, TS) TailwindCSS Firebase Auth Firestore Axios PWA (Manifest + SW)

Key Features

  • Trainer auth (email/password), middleware-protected dashboard.
  • Customer management: add (auto regNo), list, search, sort.
  • Payments per month: { "YYYY-MM": true|false }, mark paid/unpaid.
  • Dashboard due analytics: days since last payment, severity (Due/Overdue/Frozen).
  • Customer details modal with tabs: Overview, Health, Payments, Contact.
  • Payment management UI: toggle month status; total due calculation.
  • Reports and quick stats (members, active, revenue).
  • PWA install + offline cache for static assets.
  • Firestore debug page for connectivity test.

Firestore Schema (customers)

{
  name: string,
  dateOfBirth: string, // YYYY-MM-DD
  gender: 'Male' | 'Female' | 'Other',
  profilePicture?: string,
  phone: string,
  email: string,
  emergencyContactName: string,
  emergencyContactPhone: string,
  regNo: number,
  joinDate: string, // YYYY-MM-DD
  membershipType: 'Monthly' | 'Quarterly' | 'Yearly' | 'Personal Training' | 'Other',
  membershipStartDate: string,
  membershipEndDate: string,
  status: 'Active' | 'Expired' | 'On Hold' | 'Suspended',
  fee: number,
  paymentMode: 'Cash' | 'UPI' | 'Card' | 'Bank Transfer' | 'Other',
  paymentStatus: 'Paid' | 'Pending' | 'Overdue',
  dueDate: string,
  height: number,
  weight: number,
  bmi?: number,
  medicalConditions?: string,
  fitnessGoal: 'Weight Loss' | 'Muscle Gain' | 'Strength Training' | 'General Fitness' | 'Endurance' | 'Other',
  address?: { street: string, city: string, pincode: string },
  contact: string, // legacy; mirrors phone
  payments: Record<string, boolean> // { "2025-01": true, "2025-02": false }
}

Payment Logic

  • Each month key YYYY-MM = Paid/Unpaid.
  • Status rules: Current (0), Due (1â€“2), Overdue (3), Frozen (4+).
  • Due list sorts by severity then days since last payment.
  • Amount due = unpaidMonths * fee.

Error Handling & Safety

  • All Firestore calls wrapped in try/catch with [Firestore:fn] code message logs.
  • setDoc(..., { merge:true }) avoids NOT_FOUND for new docs.
  • lib/firebase.ts logs projectId in dev for quick env checks.
  • /debug/firestore page writes/reads customers/testDoc.

Setup

npm install
npm run dev

Configure .env.local with Firebase values. Visit /login, /dashboard, /dashboard/customers. Use /debug/firestore to verify Firestore.
